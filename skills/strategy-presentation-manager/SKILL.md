# 策略簡報專業經理人

像一位資深策略顧問一樣，協助您規劃、製作和優化企業策略簡報。

## 何時使用此 Skill

- 製作向性指標月報
- 規劃年度策略目標 (OKR)
- 設定與追蹤 KPI
- 分析業務數據趨勢
- 製作主管會議報告
- 策略檢討與調整建議

---

## 核心能力

### 1. 策略思維框架

**OKR 結構**
```
┌─────────────────────────────────────────────────────────────────┐
│  Object (最終目的)                                               │
│  ↓ 回答：我們要達成什麼？                                         │
├─────────────────────────────────────────────────────────────────┤
│  Goal (具體目標)                                                 │
│  ↓ 回答：可量化的里程碑是什麼？                                   │
├─────────────────────────────────────────────────────────────────┤
│  Strategy (策略)           │  Measure (衡量指標)                 │
│  ↓ 回答：如何達成？         │  ↓ 回答：如何知道成功了？            │
├────────────────────────────┼────────────────────────────────────┤
│  Plans (行動計畫)           │  Dashboard (儀表板)                │
│  ↓ 回答：具體要做什麼？      │  ↓ 回答：即時追蹤什麼數據？         │
└─────────────────────────────────────────────────────────────────┘
```

**SMART 目標檢核**
- **S**pecific（具體）：目標是否明確？
- **M**easurable（可衡量）：有量化指標嗎？
- **A**chievable（可達成）：資源足夠嗎？
- **R**elevant（相關）：與公司方向一致嗎？
- **T**ime-bound（有時限）：有明確期限嗎？

### 2. 簡報製作流程

**Phase 1：釐清需求**

一次一個問題，了解簡報目的：

```
問題清單：
1. 這份簡報的主要受眾是誰？（高層/部門主管/團隊）
2. 簡報的核心目的？（報告進度/爭取資源/策略調整）
3. 時間範圍？（月報/季報/年度）
4. 有哪些必須呈現的數據？
5. 預期的行動呼籲是什麼？
```

**Phase 2：結構設計**

先用 ASCII 呈現版面，確認後再製作：

```
向性指標簡報結構：
┌─────────────────────────────────────────────────────────────────┐
│ 投影片 1：封面                                                   │
│ ┌─────────────────────────────────────────────────────────────┐ │
│ │  2025年12月                                                 │ │
│ │  向性指標說明                                                │ │
│ │  單位：電子商務部                                            │ │
│ └─────────────────────────────────────────────────────────────┘ │
├─────────────────────────────────────────────────────────────────┤
│ 投影片 2：業務區塊標題頁 (如：廢車)                               │
├─────────────────────────────────────────────────────────────────┤
│ 投影片 3：OKR 總覽                                               │
│ ┌─────────────────────────────────────────────────────────────┐ │
│ │  Object: [最終目的]        Goal: [具體目標]                  │ │
│ ├─────────────────────────────────────────────────────────────┤ │
│ │  Strategy          │  Measure          │  Plans             │ │
│ │  ● 策略 1          │  ● 指標 1         │  ① 計畫 1          │ │
│ │  ● 策略 2          │  ● 指標 2         │  ② 計畫 2          │ │
│ │  ● 策略 3          │  ● 指標 3         │  ③ 計畫 3          │ │
│ └─────────────────────────────────────────────────────────────┘ │
├─────────────────────────────────────────────────────────────────┤
│ 投影片 4：執行狀況                                               │
│ ┌─────────────────────────────────────────────────────────────┐ │
│ │  衡量指標                                                    │ │
│ │  ┌─────────┐ ┌─────────┐ ┌─────────┐                       │ │
│ │  │ 流量    │ │ 轉換率   │ │ 成交率   │                       │ │
│ │  │ 8萬     │ │ 6.96%   │ │ 44.77%  │                       │ │
│ │  └─────────┘ └─────────┘ └─────────┘                       │ │
│ ├─────────────────────────────────────────────────────────────┤ │
│ │  策略執行                                                    │ │
│ │  ● 執行項目 1 - 進度說明                                     │ │
│ │  ● 執行項目 2 - 進度說明                                     │ │
│ └─────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────┘
```

**Phase 3：內容撰寫**

使用金字塔原理：
```
        ┌──────────────┐
        │   結論/重點   │  ← 先說結論
        └──────┬───────┘
    ┌──────────┼──────────┐
    ▼          ▼          ▼
┌───────┐ ┌───────┐ ┌───────┐
│論點 1 │ │論點 2 │ │論點 3 │  ← 支持論點
└───┬───┘ └───┬───┘ └───┬───┘
    ▼          ▼          ▼
  數據/       數據/       數據/  ← 證據支撐
  案例        案例        案例
```

**Phase 4：視覺化確認**

製作前先確認版面：
```
┌─ 確認清單 ────────────────────────────────────────────────────┐
│ □ 版面配置是否清晰？                                          │
│ □ 數據呈現方式是否易懂？                                       │
│ □ 重點是否突出？                                              │
│ □ 是否需要圖表？（趨勢圖/比較圖/佔比圖）                        │
│ □ 色彩是否符合企業風格？                                       │
└──────────────────────────────────────────────────────────────┘
```

## 工作流程

### 製作新簡報

```
1. 了解需求
   └─ 一次一個問題，確認目的、受眾、範圍

2. 收集資料
   └─ 讀取相關檔案、歷史數據

3. 設計結構
   └─ 用 ASCII 圖呈現版面，等待確認

4. 撰寫內容
   └─ 金字塔原理：結論先行，數據支撐

5. 製作簡報
   └─ 使用 document-pptx skill 或 dafong-presentation-style

6. 審核確認
   └─ 產生縮圖，確認版面無誤
```

### 分析現有簡報

```
1. 擷取內容
   └─ 使用 markitdown 轉成文字

2. 結構化整理
   └─ 分離 OKR、KPI、執行項目

3. 數據分析
   └─ 趨勢、同期比較、異常診斷

4. 提供建議
   └─ 策略調整、行動方案
```

## 常用問句

**釐清需求時：**
- 「這份報告要向誰報告？」
- 「最重要想傳達的一個訊息是什麼？」
- 「有哪些數據是必須呈現的？」
- 「希望讀者看完後採取什麼行動？」

**分析數據時：**
- 「這個數字相較上期/去年同期如何？」
- 「造成這個變化的主要原因是什麼？」
- 「這個趨勢是否符合預期？」
- 「需要採取什麼調整措施？」

**策略建議時：**
- 「目前的策略是否仍然有效？」
- 「有哪些替代方案可以考慮？」
- 「執行這個調整需要什麼資源？」
- 「預期的效益和風險是什麼？」

## 注意事項

1. **先確認再動手**：版面設計先用 ASCII 呈現，確認後再製作
2. **數據要有來源**：引用數據時標明來源和時間範圍
3. **結論要可行動**：每份報告都要有明確的 next steps
4. **格式要一致**：沿用既有的簡報格式和命名規則
5. **備份原檔**：修改既有簡報前先備份

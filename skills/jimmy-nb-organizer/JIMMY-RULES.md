## 整理經驗記錄

### 2026-01-16 H2H-Project 專案整理

**整理前**：
- 根目錄散落 20+ 個檔案
- 20 個 tmpclaude 臨時檔案
- 無統一命名規則

**整理後**：
- `docs/`：10 個文件（進度、大綱、筆記）
- `簡報/`：2 個檔案（pptx + protected pdf）
- `報告/`：3 個 PDF 報告
- `腳本/`：2 個 Python 腳本
- 根目錄：design-philosophy.md, SESSION-NOTES.md

**學到的經驗**：
1. Windows 的 `nul` 是保留字，用 `rm -f` 刪除
2. PowerShell 在 bash 環境下 `$_` 會被轉義，用腳本檔案處理
3. 日期前綴格式統一用 `YYYY-MM-DD_`（有底線）
4. `_protected.pdf` 屬於簡報的保護版，放簡報資料夾
5. 利害關係人分析的 `.md` 和 `.pdf` 放在一起（報告資料夾）

### 2026-01-16 桌面整理

**整理前**：
- 31 個資料夾散亂在桌面
- 92 個散落檔案
- 時間跨度：2017-2025（8年）

- 10 個主分類資料夾

- 28 個待決定檔案

**學到的經驗**：
1. 先建立完整資料夾結構再移動檔案
2. 合併相關資料夾可大幅減少數量
3. 年份歸檔是處理舊專案的好方法
4. 「待整理」資料夾用於暫存不確定的項目
5. 使用 `cp -r` + `rm -rf` 比直接 `mv` 更穩定（Windows 環境）

| 欄位 | 說明 | 範例 |

- `2026-01-智聯-大豐環保服務費報價.pdf`

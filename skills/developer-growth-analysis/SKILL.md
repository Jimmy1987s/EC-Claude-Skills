# 開發者成長分析

此技能通過分析您的 Claude Code 聊天互動並識別揭示優勢和成長領域的模式，提供關於您最近編碼工作的個性化反饋。

## 何時使用此技能

當您想要時使用此技能：
- 了解您最近工作的開發模式和習慣
- 識別特定的技術差距或反覆出現的挑戰
- 發現哪些主題將受益於更深入的學習
- 獲得針對您實際工作模式量身定制的精選學習資源
- 追蹤您最近專案的改進領域
- 查找直接針對您正在開發的技能的高品質文章

此技能非常適合希望獲得結構化成長反饋而無需等待程式碼審查的開發者，以及更喜歡從自己的工作歷史中獲得數據驅動洞察的開發者。

## 此技能的功能

此技能對您的開發工作執行六個步驟的分析：

1. **讀取您的聊天記錄**：訪問您本地 Claude Code 過去 24-48 小時的聊天記錄，以了解您一直在做什麼。

2. **識別開發模式**：分析您正在解決的問題類型、您正在使用的技術、您遇到的挑戰，以及您如何處理不同類型的任務。

3. **檢測改進領域**：識別表明技能差距、反覆掙扎、低效方法或您可能受益於更深入知識的領域的模式。

4. **生成個性化報告**：創建全面的報告，顯示您的工作摘要、識別的改進領域和具體的成長建議。

5. **查找學習資源**：使用 HackerNews 策劃與您的改進領域直接相關的高品質文章和討論，為您提供針對您實際開發工作量身定制的閱讀清單。

6. **發送到您的 Slack DM**：自動將完整報告傳送到您自己的 Slack 直接訊息，以便您隨時隨地參考。

## 使用方法

要求 Claude 分析您最近的編碼工作：

```
從我最近的聊天中分析我的開發者成長
```

或更具體地說明哪個時間段：

```
分析我今天的工作並建議改進領域
```

該技能將生成格式化報告，包含：
- 您最近工作的概覽
- 識別的關鍵改進領域
- 每個領域的具體建議
- 從 HackerNews 策劃的學習資源
- 您可以專注的行動項目

## 指示

當使用者請求分析他們最近工作的開發者成長或編碼模式時：

1. **訪問聊天記錄**

   從 `~/.claude/history.jsonl` 讀取聊天記錄。此檔案是 JSONL 格式，其中每行包含：
   - `display`：使用者的訊息/請求
   - `project`：正在進行的專案
   - `timestamp`：Unix 時間戳（以毫秒為單位）
   - `pastedContents`：任何貼上的程式碼或內容

   根據當前時間戳篩選過去 24-48 小時的條目。

2. **分析工作模式**

   從篩選的聊天中提取和分析以下內容：
   - **專案和領域**：使用者正在進行什麼類型的專案？（例如，後端、前端、DevOps、數據等）
   - **使用的技術**：對話中出現了哪些語言、框架和工具？
   - **問題類型**：正在解決哪些類別的問題？（例如，性能優化、除錯、功能實現、重構、設置/配置）
   - **遇到的挑戰**：使用者在哪些問題上掙扎？尋找：
     - 關於類似主題的重複問題
     - 需要多次嘗試才能解決的問題
     - 表明知識差距的問題
     - 複雜的架構決策
   - **方法模式**：使用者如何解決問題？（例如，有條理的、探索性的、實驗性的）

3. **識別改進領域**

   基於分析，識別 3-5 個使用者可以改進的具體領域。這些應該是：
   - **具體的**（不是像"改進編碼技能"這樣模糊的）
   - **基於證據的**（基於實際聊天記錄）
   - **可行的**（可以進行的實際改進）
   - **優先排序的**（最具影響力的優先）

   良好改進領域的範例：
   - "進階 TypeScript 模式（泛型、實用類型、類型守衛）- 您在 [特定專案] 中與類型安全掙扎"
   - "錯誤處理和驗證 - 我注意到您修補了幾個與缺少空值檢查相關的錯誤"
   - "Async/await 模式 - 您最近的工作顯示一些競態條件和時序問題"
   - "資料庫查詢優化 - 您多次重寫相同的查詢"

4. **生成報告**

   使用此結構創建全面報告：

   ```markdown
   # 您的開發者成長報告

   **報告期間**：[昨天 / 今天 / [自訂日期範圍]]
   **最後更新**：[當前日期和時間]

   ## 工作摘要

   [2-3 段總結使用者的工作、觸及的專案、使用的技術和整體重點領域]

   範例：
   "在過去 24 小時內，您主要專注於三個不同專案的後端開發。您的工作涉及 TypeScript、React 和部署基礎設施。您處理了功能實現、除錯和架構決策的混合，特別專注於 API 設計和資料庫優化。"

   ## 改進領域（優先排序）

   ### 1. [領域名稱]

   **為何重要**：[解釋為何這項技能對使用者的工作很重要]

   **我觀察到的**：[來自聊天記錄的顯示此差距的具體證據]

   **建議**：[改進此領域的具體步驟]

   **技能提升時間**：[所需努力的簡要估計]

   ---

   [為 2-4 個其他領域重複]

   ## 觀察到的優勢

   [2-3 個要點突出您做得好的事情 - 要繼續做的事情]

   ## 行動項目

   優先順序：
   1. [從最高優先級改進領域衍生的行動項目]
   2. [從下一個領域的行動項目]
   3. [從下一個領域的行動項目]

   ## 學習資源

   [將在下一步填充]
   ```

5. **搜索學習資源**

   使用 Rube MCP 搜索 HackerNews 中與每個改進領域相關的文章：

   - 對於每個改進領域，構建針對高品質資源的搜索查詢
   - 使用 RUBE_SEARCH_TOOLS 搜索 HackerNews，查詢如：
     - "Learn [Technology/Pattern] best practices"
     - "[Technology] advanced patterns and techniques"
     - "Debugging [specific problem type] in [language]"
   - 優先考慮高參與度的帖子（評論、投票）
   - 對於每個領域，包括 2-3 篇最相關的文章，包含：
     - 文章標題
     - 發布日期
     - 為何相關的簡要描述
     - 文章連結

   將此部分添加到報告中：

   ```markdown
   ## 精選學習資源

   ### 用於：[改進領域]

   1. **[文章標題]** - [日期]
      [它涵蓋的內容以及為何與您的改進領域相關的描述]
      [連結]

   2. **[文章標題]** - [日期]
      [描述]
      [連結]

   [為其他改進領域重複]
   ```

6. **呈現完整報告**

   以乾淨、可讀的格式提供報告，使用者可以：
   - 快速掃描關鍵要點
   - 用於專注學習規劃
   - 在接下來的一週工作改進時參考
   - 如果他們想要外部反饋，可以與導師分享

7. **將報告發送到 Slack DM**

   使用 Rube MCP 將完整報告發送到使用者自己的 Slack DM：

   - 通過 RUBE_SEARCH_TOOLS 檢查 Slack 連接是否活躍
   - 如果未連接，使用 RUBE_MANAGE_CONNECTIONS 啟動 Slack 授權
   - 使用 RUBE_MULTI_EXECUTE_TOOL 將報告作為格式化訊息發送：
     - 將報告標題和期間作為第一條訊息發送
     - 將報告分解為邏輯部分（摘要、改進、優勢、行動、資源）
     - 將每個部分格式化為具有適當 markdown 的結構良好的 Slack 訊息
     - 包括學習資源的可點擊連結
   - 在 CLI 輸出中確認交付

   這確保使用者在他們定期檢查的地方擁有報告，並可以在整週內參考它。

# 您的開發者成長報告

## 工作摘要

在過去兩天中，您專注於後端基礎設施和 API 開發。您的主要專案是一個開源展示應用程式，在其中您在連接管理、UI 改進和部署配置方面取得了重大進展。您使用 TypeScript、React 和 Node.js，處理從數據安全到響應式設計的挑戰。您的工作顯示了實現功能和解決技術債務之間的平衡。

## 改進領域（優先排序）

### 1. 進階 TypeScript 模式和類型安全

**為何重要**：TypeScript 是您工作的核心，但利用其進階功能（泛型、實用類型、條件類型、類型守衛）可以顯著提高程式碼可靠性並減少運行時錯誤。更好的類型安全在編譯時而非在生產中捕獲錯誤。

**我觀察到的**：在您最近的聊天中，您正在處理連接數據結構，並在正確鍵入授權配置方面掙扎了幾次。您還必須迭代不同連接狀態的聯合類型。有機會更有效地使用區分聯合和類型守衛。

**建議**：學習 TypeScript 的進階類型系統，特別是實用類型（Omit、Pick、Record）、條件類型和區分聯合。將這些模式應用於您的連接配置處理和授權狀態管理。

**技能提升時間**：5-8 小時的專注學習和練習

### 2. UI 中的安全數據處理和資訊隱藏

**為何重要**：您識別並修復了一個安全問題，其中敏感連接數據在您的控制台中顯示。防止資訊洩漏對於處理使用者憑證和 API 金鑰的應用程式至關重要。這裡的良好實踐可防止安全事件和使用者信任違規。

**我觀察到的**：您發現您的"您的應用"頁面顯示了完整的連接數據，包括授權配置。這顯示了良好的安全本能，下一步是將此構建到您處理敏感資訊時的預設思維中。

**建議**：查看前端應用程式中處理敏感數據的安全最佳實踐。創建可重用模式，用於在顯示之前過濾/遮罩敏感資訊。考慮實現一個安全數據層，明確將可以在 UI 中顯示的內容列入白名單。

**技能提升時間**：3-4 小時

### 3. 元件架構和響應式 UI 模式

**為何重要**：您正在設計需要在不同螢幕尺寸和使用者互動中工作的 UI。強大的元件架構使構建複雜 UI 更容易，沒有錯誤，並提高可維護性。

**我觀察到的**：您在"Marketplace" UI（以前稱為 Browse Tools）上工作，從設計圖像重新創建它。您還識別並修復了內容溢出容器的滾動問題。有機會加強您對佈局包含和響應式設計模式的理解。

**建議**：學習 React 元件組合模式和 CSS 佈局最佳實踐（特別是 flexbox 和 grid）。專注於防止溢出問題的容器查詢和響應式模式。研究元件組合庫和設計系統方法。

**技能提升時間**：6-10 小時（取決於深度）

## 觀察到的優勢

- **安全意識**：您在數據洩漏問題成為問題之前主動識別它們
- **迭代改進**：您有條理地處理 UI 要求，提出澄清問題並改進設計
- **全端能力**：您可以輕鬆地跨後端 API、前端 UI 和部署問題工作
- **解決問題的方法**：您將複雜任務分解為可管理的步驟

## 行動項目

優先順序：
1. 花 1-2 小時學習 TypeScript 實用類型和區分聯合；應用於您的連接數據結構
2. 為您的專案記錄安全模式（哪些數據可以安全顯示、過濾/遮罩函數）
3. 學習一篇關於進階 React 模式的文章，並將一個模式應用於您當前的 UI 工作
4. 為未來的 PR 設置一個專注於類型安全和數據安全的程式碼審查檢查清單

## 技巧和最佳實踐

- 每週運行一次此分析以追蹤您隨時間的改進軌跡
- 一次選擇一個改進領域並專注於它幾天，然後再轉到下一個
- 使用學習資源作為學習指南；研究推薦的材料並練習應用模式
- 在專注於一個領域一週後重新訪問此報告，以查看您的工作模式如何變化
- 學習資源是專門為您的實際工作策劃的，而不是通用主題，因此它們與您正在構建的內容高度相關

## 如何保持準確性和質量

此技能：
- 從帶時間戳的聊天記錄中分析您的實際工作模式
- 生成基於真實專案的基於證據的建議
- 策劃直接針對您識別的差距的學習資源
- 專注於可行的改進，而非模糊的反饋
- 根據複雜性提供具體的時間估計
- 優先考慮對您的開發速度影響最大的領域

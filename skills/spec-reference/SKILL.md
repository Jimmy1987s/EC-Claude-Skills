# 開發規格參考系統

此 skill 管理專案的技術規格文件，每個功能的規格存放在獨立檔案中。

## 目錄結構

```
~/.claude/skills/spec-reference/
├── SKILL.md          # 主檔（本檔案）- 索引與使用說明
└── specs/            # 規格檔案目錄
    ├── image-upload.md
    ├── [其他規格].md
    └── ...
```

## 規格索引

| 規格名稱 | 檔案 | 狀態 | 最後更新 |
|----------|------|------|----------|
| 圖片上傳與壓縮 | [image-upload.md](specs/image-upload.md) | ✅ | 2026-01-16 |

## 使用方式

### 查詢規格
- 「查一下圖片上傳的規格」→ 讀取對應的 specs/*.md
- 「有哪些規格？」→ 查看上方索引表

### 新增規格
說「把 XXX 功能的規格加到 spec-reference」，我會：
1. 在 `specs/` 建立新檔案
2. 更新索引表

## 規格檔案範本

新增規格時，使用以下格式：

```markdown
# [功能名稱]規格

## 概述
（一句話說明此功能）

## 限制條件
| 項目 | 限制值 | 說明 |
|------|--------|------|
| ... | ... | ... |

## 實作流程
（流程圖或步驟說明）

## 關鍵程式碼
- [檔案名稱](相對路徑)

## 注意事項
- （踩過的坑）
- （特殊處理）

## 變更記錄
| 日期 | 變更內容 |
|------|----------|
| YYYY-MM-DD | 初版 |
```

# 潛在客戶研究助手

此技能透過分析您的產品/服務、了解您的理想客戶輪廓，並提供可行的外聯策略，幫助您識別和評估潛在客戶。

## 何時使用此技能

- 為您的產品/服務尋找潛在客戶或客戶
- 建立要聯繫的公司清單以建立合作夥伴關係
- 識別銷售外聯的目標帳戶
- 研究符合您理想客戶輪廓的公司
- 準備業務開發活動

## 此技能的功能

1. **了解您的業務**：分析您的產品/服務、價值主張和目標市場
2. **識別目標公司**：根據以下條件尋找符合您理想客戶輪廓的公司：
   - 產業和部門
   - 公司規模和地點
   - 他們使用的技術堆疊和工具
   - 成長階段和資金
   - 您的產品解決的痛點
3. **優先排序潛在客戶**：根據適配分數和相關性對公司進行排名
4. **提供聯繫策略**：建議如何以個性化訊息接近每個潛在客戶
5. **豐富資料**：收集有關決策者和公司背景的相關資訊

## 如何使用

### 基本用法

只需描述您的產品/服務和您正在尋找的內容：

```
我正在建立 [產品描述]。為我找到 [地點/產業] 中 10 家公司，
這些公司會是很好的潛在客戶。
```

### 使用您的程式碼庫

為了獲得更好的結果，從您產品的原始碼目錄運行此命令：

```
查看我在此存放庫中建立的內容，並識別 [地點/產業] 中
前 10 家會從此產品中受益的公司。
```

### 進階用法

對於更有針對性的研究：

```
我的產品：[描述]
理想客戶輪廓：
- 產業：[產業]
- 公司規模：[規模範圍]
- 地點：[地點]
- 目前的痛點：[痛點]
- 他們使用的技術：[技術堆疊]

為我找到 20 個合格的潛在客戶，並為每個提供聯繫策略。
```

## 操作說明

當使用者請求潛在客戶研究時：

1. **了解產品/服務**
   - 如果在程式碼目錄中，分析程式碼庫以了解產品
   - 詢問有關價值主張的澄清問題
   - 識別關鍵功能和優勢
   - 了解它解決了什麼問題

2. **定義理想客戶輪廓**
   - 確定目標產業和部門
   - 識別公司規模範圍
   - 考慮地理偏好
   - 了解相關痛點
   - 註明任何技術要求

3. **研究和識別潛在客戶**
   - 搜尋符合條件的公司
   - 尋找需求信號（職位發布、技術堆疊、最新新聞）
   - 考慮成長指標（資金、擴張、招聘）
   - 識別具有互補產品/服務的公司
   - 檢查預算指標

4. **優先排序和評分**
   - 為每個潛在客戶建立適配分數（1-10）
   - 考慮以下因素：
     - 與理想客戶輪廓的一致性
     - 立即需求的信號
     - 預算可用性
     - 競爭格局
     - 時機指標

5. **提供可行的輸出**

   對於每個潛在客戶，提供：
   - **公司名稱**和網站
   - **為什麼他們是好選擇**：基於其業務的具體原因
   - **優先級分數**：1-10 並附上解釋
   - **決策者**：要針對的角色/職稱（例如，"工程副總裁"）
   - **聯繫策略**：個性化方法建議
   - **價值主張**：您的產品如何解決他們的特定問題
   - **對話開場白**：在外聯中提及的具體要點
   - **LinkedIn URL**：如果有，便於連接

6. **格式化輸出**

   以清晰、可掃描的格式呈現結果：

   ```markdown
   # 潛在客戶研究結果

   ## 摘要
   - 找到的潛在客戶總數：[X]
   - 高優先級（8-10）：[X]
   - 中優先級（5-7）：[X]
   - 平均適配分數：[X]

   ---

   ## 潛在客戶 1：[公司名稱]

   **網站**：[URL]
   **優先級分數**：[X/10]
   **產業**：[產業]
   **規模**：[員工人數/營收範圍]

   **為什麼他們是好選擇**：
   [基於其業務的 2-3 個具體原因]

   **目標決策者**：[角色/職稱]
   **LinkedIn**：[如果有則提供 URL]

   **為他們提供的價值主張**：
   [此公司的具體優勢]

   **外聯策略**：
   [個性化方法 - 提及具體痛點、最近的公司新聞或相關背景]

   **對話開場白**：
   - [具體要點 1]
   - [具體要點 2]

   ---

   [對每個潛在客戶重複]
   ```

7. **提供下一步**
   - 建議將結果儲存為 CSV 以匯入 CRM
   - 提供起草個性化外聯訊息
   - 根據時機推薦優先排序
   - 建議對頂級潛在客戶進行後續研究

## 範例

### 範例 1：來自 Lenny's Newsletter

**使用者**："我正在建立一個工具，可以在 AI 程式設計助手查詢中遮罩敏感資料。尋找潛在客戶。"

**輸出**：建立一個優先排序的公司清單，這些公司：
- 使用 AI 程式設計助手（Copilot、Cursor 等）
- 處理敏感資料（金融科技、醫療保健、法律）
- 在其 GitHub 存放庫中有使用程式設計代理的證據
- 可能在程式碼中意外暴露了敏感資料
- 包括相關決策者的 LinkedIn URL

### 範例 2：本地業務

**使用者**："我經營一家遠端團隊生產力諮詢業務。為我在灣區找到 10 家最近轉為遠端的公司。"

**輸出**：識別以下公司：
- 最近發布了遠端職位清單
- 宣布了遠端優先政策
- 正在招聘分散式團隊
- 顯示出遠端工作挑戰的跡象
- 為每個提供個性化的外聯策略

## 獲得最佳結果的提示

- **具體說明**您的產品及其獨特價值
- **從您的程式碼庫運行**（如果適用）以獲得自動背景
- **提供背景**有關您理想客戶輪廓的資訊
- **指定約束條件**如產業、地點或公司規模
- **請求後續**對有前景的潛在客戶進行更深入的研究
